# AI Genomics Paper Daily Feed

A Python-based literature monitoring tool that automatically finds recent papers related to AI, machine learning, genetics, genomics, and computational biology, then sends one organized email digest.

The goal is to help researchers stay updated on both:

1. emerging AI methods for genomics, and
2. practical machine learning/statistical methods useful for genetics research.

## Overview

This project searches recent papers from multiple sources, filters them using topic keywords and heuristic scoring, groups them into meaningful sections, and emails a daily HTML digest.

The combined digest is generated by:

```text
genetics_paper_digest.py
```

The email is organized into sections such as:

```text
Emerging AI + Genomics
Practical ML / Genetics Methods
Journal Watch
Other Relevant Papers
```

## Topics covered

The digest is designed to track papers related to:

* Genomic foundation models

* Single-cell AI and cell foundation models

* GWAS, eQTL, fine-mapping, and statistical genetics

## Setup

### 1. Clone the repository


```bash
git clone https://github.com/Jihua-Liu/genetics-paper-digest.git
cd genetics-paper-digest
```

### 2. Create a conda environment

```bash
conda create -n paper_digest python=3.11 -y
conda activate paper_digest
pip install -r requirements.txt
```

Alternatively, use any Python 3.10+ environment:

```bash
pip install -r requirements.txt
```

### 3. Create a `.env` file

Create a `.env` file in the repository folder:

```bash
EMAIL_SENDER=your_sender_email@gmail.com
EMAIL_PASSWORD=your_gmail_app_password
EMAIL_RECEIVER=your_receiver_email@example.com
```

For Gmail, use a Gmail App Password instead of your normal Gmail password.

To send to multiple recipients, separate email addresses with commas:

```bash
EMAIL_RECEIVER=first_email@example.com,second_email@example.com,third_email@example.com
```

The `.env` file should not be committed to GitHub. It is ignored by `.gitignore`.

## Local testing

After creating the environment and `.env` file, test manually:

```bash
python genetics_paper_digest.py
```

If the setup works, you should receive one organized email digest.

## Run automatically on a server with cron

This is the recommended option if you have access to a lab server that stays on.

Example project path:

```bash
/PATHTOPROJECT/genetics-paper-digest
```

Example conda Python path:

```bash
/PATHTOCONDA/anaconda3/envs/paper_digest/bin/python
```

Edit crontab:

```bash
crontab -e
```

Add this line to run the combined digest every day at 8:00 AM server time:

```bash
0 8 * * * cd /PATHTOTHEPROJECT/genetics-paper-digest && /PATHTOTHECONDAENV/anaconda3/envs/paper_digest/bin/python genetics_paper_digest.py >> genetics_paper_digest.log 2>&1
```

Check your cron jobs:

```bash
crontab -l
```

Check logs:

```bash
tail -50 genetics_paper_digest.log
```

## Run with GitHub Actions

GitHub Actions can also run the digest on GitHub-hosted runners.

Add these repository secrets:

```text
EMAIL_SENDER
EMAIL_PASSWORD
EMAIL_RECEIVER
```

In GitHub:

```text
Repository → Settings → Secrets and variables → Actions → New repository secret
```

The workflow file is located at:

```text
.github/workflows/daily_digest.yml
```

Example daily schedule:

```yaml
on:
  schedule:
    - cron: "20 14 * * *"
  workflow_dispatch:
```

GitHub Actions cron uses UTC time. During daylight saving time in Madison/Chicago, `14:20 UTC` is `9:20 AM` Central Time.

## Server vs GitHub Actions

| Option            | Pros                                                                   | Cons                                                             |
| ----------------- | ---------------------------------------------------------------------- | ---------------------------------------------------------------- |
| Lab server + cron | Reliable if server is always on; easy local logs; uses `.env` directly | Depends on server permissions and SMTP access                    |
| GitHub Actions    | Does not depend on laptop/server; easy manual runs                     | Requires GitHub Secrets; scheduled runs can sometimes be delayed |
| Laptop + cron     | Simple for testing                                                     | Fails if laptop is asleep or closed                              |

For regular use, a lab server or GitHub Actions is better than laptop cron.

## Customization

You can customize the digest by editing keyword lists and scoring rules in:

```text
genetics_paper_digest.py
```

Common settings to adjust include:

```python
DAYS_BACK
MAX_PAPERS_PER_EMAIL
MIN_QUALITY_SCORE
TARGET_JOURNALS
PUBMED_TOPIC_TERMS
```

Lowering `MIN_QUALITY_SCORE` will include more papers but may increase noise. Raising it will make the digest stricter.

## Disclaimer

This digest uses keyword filtering and heuristic scoring. It is intended for literature monitoring and paper discovery, not for systematic review or complete literature coverage.
